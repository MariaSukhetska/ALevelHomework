HTTP (Hypertext Transfer Protocol) є протоколом передачі гіпертексту, який використовується для обміну даними
між клієнтом і сервером у веб-сервісах. Основна ідея HTTP полягає в тому, щоб клієнт і сервер могли обмінюватися
повідомленнями у вигляді текстових даних.
GET-запит - це один із методів HTTP-протоколу, який використовується для отримання (зчитування) даних з вказаного
ресурсу на сервері. GET-запит використовується для отримання інформації, і він не повинен змінювати стану сервера
або ресурсу.
Cтруктура GET-запиту:
GET /path/to/resource?param1=value1&param2=value2 HTTP/1.1
Host: example.com
1. Лінія запиту (Request Line):
   - Метод: GET
   - Цільовий ресурс (URL): /path/to/resource?param1=value1&param2=value2
   - Версія протоколу: HTTP/1.1
2. Заголовок (Host):
   - Вказує ім'я хоста (сервера), до якого надсилається запит.
   - Приклад: Host: example.com
Це базова структура, яка включає в себе лінію запиту та обов'язковий заголовок Host.
Інші заголовки та параметри додаються за потреби.
POST-запит є одним з методів HTTP-протоколу, призначеним для відправки даних на сервер для обробки.
POST використовується для надсилання даних на сервер з метою збереження чи обробки даних.
Основні риси POST-запиту:
1. Лінія запиту (Request Line):
   - Метод: POST
   - Цільовий ресурс (URL): /path/to/resource
   - Версія протоколу: HTTP/1.1
2. Заголовки (Headers):
   - Content-Type: вказує тип медіа-формату в тілі запиту (наприклад, application/json, `application/x-www-form-urlencoded`).
   - Content-Length: вказує довжину тіла запиту в байтах.
   - Інші заголовки за потреби.
3. Тіло (Body):
   - Фактичні дані, які відправляються на сервер.
4. Пустий рядок (CRLF):
   - Після тіла запиту слідує порожній рядок, що вказує на завершення POST-запиту.
Приклад POST запиту з JSON тілом
POST /api/user/create HTTP/1.1
Host: example.com
Content-Type: application/json
{
"username": "Maria",
"email": "maria.zhdanova1234@gmail.com",
"password": "security12345"
}
